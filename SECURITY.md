# Security Policy

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Report security issues by email to **info@oxigate.com**. Include:

- A description of the vulnerability and its potential impact
- Steps to reproduce or proof-of-concept (if available)
- Affected versions or components

You will receive an acknowledgement within 5 business days. We aim to
provide a resolution timeline within 14 days of triage.

## Scope

In scope: the OxiGate gateway binary, its configuration, and its API surface.

Out of scope: third-party provider APIs, infrastructure you operate, and
issues in upstream dependencies (report those to the upstream project directly;
we will coordinate if OxiGate is a required vector).

## Disclosure Policy

We follow coordinated disclosure. Please allow reasonable time for a fix to be
prepared and released before any public disclosure.
